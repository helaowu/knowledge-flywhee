# 知识复利系统 — Claude Code 配置

## 实时观测记录（raw_log）

对话中遇到以下情况，立即 append 到 `memory/raw_log.md`：

① 用户纠正错误
② 确认的定义/口径/规则
③ 发现可复用的模式或流程
④ 用户明确表达偏好
⑤ 用户明确认可某个做法（正向强化）

### 写入格式

```markdown
## [YYYY-MM-DD] | ①类型标签 | 标题
- 内容：具体发生了什么
- 出现次数：1
- 升级候选：（目标文件路径）
- 状态：raw
```

### 规则
- 已有同类条目 → 更新出现次数，不新建
- 出现次数 ≥ 3 → 下次 review-memory 必须升级为正式规则
- 每轮回复结束后，主动判断是否应触发记录

## 知识库自动维护（Wiki）

每次任务完成后，检查以下四项（有就写，没有就跳过）：

1. **新概念/数据源** → 补充到知识库对应页面
2. **确认的定义/口径** → 追加到相关页面
3. **踩坑记录** → 写入 `wiki/lessons/`
4. **可复用模式** → 写入 `wiki/patterns/`

## 升级规则

### 收录阈值
- 这个行为 AI 在类似上下文里会漏掉吗？会漏 → 收录
- 一次性边界情况 → 不收录

### 升级检查项
1. 不重叠：先读现有内容，有重叠的更新而不是新增
2. 能泛化：能不提触发它的具体任务写出来吗？
3. 能执行：不是"更仔细"，是"做 X"或"不做 Y"
4. 放对位置：领域规则 → conventions；行为 → CLAUDE.md；经验 → memory

## 更新系统文件的约束

更新任何 memory、conventions、CLAUDE.md 之前，必须：
1. 展示改动内容给用户
2. 等用户确认
3. 才能执行修改
