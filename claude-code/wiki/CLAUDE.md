# Wiki 维护规则

## 自动维护触发

每次任务完成后，检查以下四项：

### 1. 新概念/数据源
本次使用了一个之前没记录过的概念或数据源 → 新建或补充页面

### 2. 定义确认
本次确认了一个之前不确定的定义/口径/规则 → 追加到对应页面

### 3. 踩坑记录
本次发现了一个陷阱/Bug/反直觉行为 → 追加到 `lessons/lessons.md`

### 4. 可复用模式
本次写了有通用价值的代码/查询/流程 → 追加到 `patterns/patterns.md`

## 页面格式

### 概念/数据源页面

```markdown
# [名称]

## 基本信息
- 一句话描述
- 关键字段/属性

## 使用注意
- 坑点
- 常见误用

## 典型用法
- 示例代码/查询
```

### Lessons 格式

```markdown
## [日期] 标题
- 现象：发生了什么
- 原因：为什么
- 正确做法：应该怎么做
```

### Patterns 格式

```markdown
## [标题]
- 场景：什么时候用
- 模式：代码/查询/流程
- 注意：边界条件
```

## 原则

- 不重复记录：先搜索有没有已有页面
- 一个概念一个位置：不在多个地方写同一个东西
- 新建页面后更新 index.md
