# Review Memory

审查 raw_log，把够格的条目升级为正式规则。

## 触发方式

用户调用 `/review-memory` 时执行。

## 执行步骤

### 1. 读取 raw_log.md

找出所有条目，重点关注：
- 出现次数 ≥ 3 的（必须升级）
- 状态仍为 `raw` 的

### 2. 逐条审查

对每个候选条目，过以下检查：

**收录检查**：
- [ ] 不重叠：现有规则里有没有类似的？有 → 合并更新
- [ ] 能泛化：能脱离具体任务来表述吗？不能 → 重写
- [ ] 能执行：是"做 X / 不做 Y"还是"更仔细"？后者不收录
- [ ] 放对位置：
  - 领域规则 → conventions 文件
  - 行为规则 → CLAUDE.md
  - 项目事实 → memory 文件
  - 技术知识 → wiki

### 3. 执行升级

对通过检查的条目：
1. 写出新规则的具体内容
2. 展示给用户确认
3. 确认后写入目标文件
4. 更新 raw_log 中该条目的状态为 `upgraded`

### 4. 清理

- 删除已过时的条目
- 合并重复条目
- 更新 MEMORY.md 索引（如果新增了 memory 文件）

### 5. 报告

输出本次审查结果：
- 升级了 N 条（列出标题 + 目标位置）
- 删除了 N 条过时条目
- 合并了 N 条重复条目
- 剩余 N 条待观察（出现次数 < 3）
