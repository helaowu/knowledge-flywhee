# 6 个复利机制详解

## 1. 错题集（raw_log）

### 是什么
一个 append-only 的日志文件，记录对话中发生的关键事件。

### 触发条件
对话中出现以下情况时自动写入：
- 用户纠正了 AI 的错误
- 确认了一个之前不确定的定义/口径
- 发现了可复用的模式
- 用户表达了明确的偏好
- 用户明确认可了某个做法（正向强化）

### 格式

```markdown
## [2026-01-15] | 纠正 | 分区字段不能套函数
- 内容：用户纠正"where substr(dt,1,4) >= '2026'"会导致全表扫描，应直接比较字面量
- 出现次数：1
- 升级候选：sql_conventions.md
- 状态：raw
```

### 关键规则
- 已有同类条目 → 更新出现次数，不新建
- 出现次数 ≥ 3 → 必须升级为正式规则

## 2. 3 次升级规则

### 是什么
raw_log 中同一类问题被记录 3 次后，自动（或通过 review-memory 命令）升级为正式规则。

### 升级目标
- 行为规则 → CLAUDE.md
- 领域规则 → conventions 文件
- 项目事实 → memory 文件
- 技术知识 → wiki/知识库

### 升级检查项
1. 不重叠：先读现有规则，有重叠的更新而不是新增
2. 能泛化：能脱离触发它的具体任务来表述吗？
3. 能执行："做 X"或"不做 Y"，不是"更仔细"
4. 放对位置：领域 → conventions；行为 → CLAUDE.md；经验 → memory

---

## 3. 笔记本自动生长（Wiki）

### 是什么
一个本地知识库，每次任务完成后自动检查是否有新东西值得记录。

### 自动检查项（每次任务后）
1. 碰到新的数据源/工具/概念 → 记录说明
2. 确认了一个定义或口径 → 写入对应页面
3. 发现了坑/陷阱 → 写入 lessons
4. 写了有复用价值的代码/查询 → 写入 patterns

### 笔记本结构

```
wiki/
├── index.md          ← 页面目录
├── lessons/          ← 踩坑记录
│   └── lessons.md
├── patterns/         ← 可复用模式
│   └── patterns.md
└── [领域特定页面]
```

## 4. 压缩前存档（pre_compact_hook）

### 是什么
在 AI 对话上下文被压缩之前，自动提取关键信息存入文件系统。

### 触发时机
对话接近上下文窗口限制，系统即将执行压缩/摘要时。

### 保存内容
- 当前进行中的任务状态
- 本次对话中确认的重要决定
- 尚未完成的待办事项
- 关键的中间结果

### 为什么重要
LLM 的上下文压缩会丢信息。没有这个机制，长对话后半段会"忘记"前半段确认的东西。

## 5. 流程沉淀（Skill/Command）

### 是什么
完成一个非平凡任务后，判断"下次还会做吗"。会的话存成一键命令。

### 触发条件
任务满足以下任一：
- 步骤 > 3 步
- 涉及特定工具组合
- 用户明确说"以后还会做"
- 是定期任务（周报、月报）

### 存储格式
一个 markdown 文件，包含：
- 触发词（什么时候用）
- 前置条件（需要什么输入）
- 执行步骤
- 输出格式

---

## 6. 审查+升级流程（review-memory）

### 是什么
定期审查 raw_log，把够格的条目升级为正式规则。

### 触发方式
- 手动：用户调用 `/review-memory`
- 自动：当 raw_log 中某条目出现次数 ≥ 3 时，下次交互主动提醒升级

### 审查检查项
1. 哪些条目出现 ≥ 3 次？→ 必须升级
2. 哪些条目已过时？→ 删除
3. 有没有重复/矛盾的条目？→ 合并
4. 升级后的规则放对位置了吗？

### 输出
- 升级了哪些条目（从 raw → 正式规则）
- 删除了哪些过时条目
- 合并了哪些重复条目
